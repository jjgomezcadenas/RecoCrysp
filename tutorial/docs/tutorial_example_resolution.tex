\begin{abstract}
A second worked example: a Derenzo rod phantom in a water cylinder on the
continuous (CRYSP) scanner. To separate the reconstruction from the resolution we
simulate \emph{two} datasets that differ only by the detector blur --- one sharp
($\Gop=\mathbf I$), one smeared ($\Gop$ a $5\,\mathrm{mm}$ Gaussian) --- and
reconstruct both with the same attenuation-corrected MLEM. The sharp data
recover the sharp rods (validating the reconstruction); the smeared data recover
the resolution-limited image $\Gop\,\xv$. Their difference is purely the
resolution. Code in \code{tutorial/examples/resolution/}.
\end{abstract}

\section{Overview}
\label{sec:ex-res:overview}

Resolution is a Gaussian point-spread $\Gop$ applied to the activity (Part~I,
Detector Resolution), injected \emph{at simulation time}: the reconstruction runs
with the plain projector $\Aop$ and converges towards $\Gop\,\xv$. To show this
cleanly --- and to check that the reconstruction itself is faithful --- we run it
twice on the same geometry, attenuation and sensitivity, changing only the
activity that generates the data:
\begin{equation}
  \text{sharp:}\quad \pred = a\,(\Aop\,\xv) ,
  \qquad
  \text{smeared:}\quad \pred = a\,(\Aop\,\Gop\,\xv) ,
  \label{eq:ex-res-two}
\end{equation}
with $a_i=\exp(-\!\int_{\mathrm{LOR}_i}\mu\,\mathrm{d}\ell)$. A Derenzo phantom
--- sectors of rods at decreasing diameter --- makes $\Gop$ unmistakable. We
exaggerate the resolution to $5\,\mathrm{mm}$ for clarity (CRYSP's intrinsic
value is $3.5\,\mathrm{mm}$; a one-line change in the configuration).

\section{Setup}
\label{sec:ex-res:setup}

The scanner is the CRYSP continuous cylinder
($\varnothing\,77.4\,\mathrm{cm}$, AFOV $102.4\,\mathrm{cm}$). The phantom is a
water cylinder of radius $80\,\mathrm{mm}$ --- the $\mu$-map --- with six sectors
of rods of diameters $\{3,4,5,6,8,10\}\,\mathrm{mm}$. To render the rods and a
sharp reconstruction the grid is fine, $1\,\mathrm{mm}$ isotropic
($181\times181\times101$). Both phantom and grid come from the library:
\begin{lstlisting}
x_true = derenzo(n, org, vs; radius = 80.0, length = 100.0,
                 rod_diameters = [3,4,5,6,8,10], spacing = 2.0)
mumap  = uniform_cylinder(n, org, vs; radius = 80.0, length = 100.0,
                          value = 0.0096)   # water, mm^-1
x_blur = gaussian_blur(x_true, 5.0, vs)     # G x_true (the recoverable image)
\end{lstlisting}

\section{Simulation and reconstruction}
\label{sec:ex-res:sim}

The attenuation factor $\att$ and the sensitivity $\sensv=\Aadj(\att)$ depend
only on the geometry and the $\mu$-map, so they are computed once and shared by
both datasets. Each dataset is then a Poisson realization of \eqref{eq:ex-res-two}
and is reconstructed with the same attenuation-corrected, plain-$\Aop$ MLEM:
\begin{lstlisting}
a    = exp.(-joseph3d_fwd(xs, xe, mumap, org, vs))
sens = sensitivity_image(xs, xe, n, org, vs; weights = a)

function simulate_reconstruct(activity, seed_off)   # a, sens, xs, xe shared
    ybar  = a .* joseph3d_fwd(xs, xe, activity, org, vs)
    counts = [rand(rng, Poisson(lam)) for lam in scale .* ybar]
    model  = ListmodePoissonModel(xs, xe, sens; img_origin = org, voxsize = vs,
                                  counts = counts, mult = a)
    return mlem(model, x0; niter = 50)
end
rec_sharp = simulate_reconstruct(x_true, 1)    # G = 1 data
rec_blur  = simulate_reconstruct(x_blur, 2)    # G = 5 mm data
\end{lstlisting}

\section{Results}
\label{sec:ex-res:results}

Figure~\ref{fig:ex-res} shows the four central slices. The reconstruction from
the \emph{sharp} data (third panel) recovers the truth: every rod sector,
including the finest, comes back, with a rod/gap contrast of $40$ --- the only
residual is the $\sim\!1\,\mathrm{mm}$ intrinsic blur of the Joseph
interpolation, and the contrast keeps climbing with iterations (it was $15$ at
$20$ iterations, $40$ at $50$), confirming the reconstruction converges to the
truth. The reconstruction from the \emph{smeared} data (fourth panel) instead
recovers $\Gop\,\xv$ (second panel): its rod/gap contrast is $4.3$, matching the
recoverable value of $4.5$, so the coarse sectors resolve and the
$3$--$4\,\mathrm{mm}$ sectors merge at the $5\,\mathrm{mm}$ limit. The two
reconstructions use \emph{identical} code and settings; their difference is
entirely the detector resolution.

  \centering

  \caption{Derenzo phantom, central slice. From left: the sharp truth; the
    resolution-limited image $\Gop\,\xv$; the MLEM reconstruction from sharp
    ($\Gop=\mathbf I$) data, which recovers the truth; and the MLEM
    reconstruction from smeared ($5\,\mathrm{mm}$) data, which recovers
    $\Gop\,\xv$. Identical reconstruction in both --- only the data differ.}

A practical note on sampling. The cleanliness here needs many LORs, not many
counts: a continuous scanner's randomly sampled LORs cover line space far less
uniformly than a structured sinogram, so resolving fine structure requires dense
angular coverage --- $10^8$ LORs in this run. Fewer LORs leave the rods grainy
\emph{however many counts} are collected; the cure is more LORs.

\section{Running the example}
\label{sec:ex-res:run}

The code is in \code{tutorial/examples/resolution/}: \code{run.jl},
\code{resolution.toml} (all parameters), \code{resolution\_plot.py}. From the
repository root:
\begin{lstlisting}
julia --project=tutorial/examples/resolution -e \
  'using Pkg; Pkg.develop(path="."); Pkg.instantiate()'   # first time
julia -t auto --project=tutorial/examples/resolution \
  tutorial/examples/resolution/run.jl
python3 tutorial/examples/resolution/resolution_plot.py
\end{lstlisting}
The run is deliberately heavy ($10^8$ LORs, $1\,\mathrm{mm}$ voxels, two
reconstructions, a few minutes); lower \code{n\_lors} or raise \code{voxsize} in
the TOML to run lighter, or set \code{fwhm} to $3.5\,\mathrm{mm}$ for the true
CRYSP resolution.

\section{What this leaves out}
\label{sec:ex-res:next}

Attenuation and resolution are both in place. The remaining physical effects ---
\emph{randoms} and \emph{scatter} --- are additive backgrounds entering through
the \code{contamination} field, added in the following examples without changing
the reconstruction code, only the simulated data (Part~I, The Complete Data
Model)~\cite{crysp2026}.
